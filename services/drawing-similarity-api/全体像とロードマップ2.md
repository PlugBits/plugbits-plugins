現時点のトライアルでは、UI より先に検索品質を固める。

# 目標

kintone 上で図面を正本にしつつ、

```text
類似図面検索
図面登録支援
```

を段階的に実装する。

---

# いま確実にできること

```text
PDF取得
↓
1ページ目をPNG化
↓
OpenCLIPでEmbedding
↓
Qdrant登録
↓
類似検索
```

---

# いまの判断

OpenCLIP だけでは、図面全体の見た目の近さは取れても、部品形状の近さには寄り切らない。

そのため、当面は次の順で品質を上げる。

```text
OCR
項目抽出
OpenCLIP補助
形状特徴
```

---

# 今回の実装範囲

## フェーズ1

類似検索を安定して使える状態にする。

やること:

```text
OpenCLIP入力画像の crop 切替
OCR の追加
処理ログの可視化
timeout 制御
Qdrant payload の拡張
```

完了条件:

```text
100件程度を登録できる
検索が止まらない
Cloud Run ログで停止箇所が追える
```

## フェーズ2

図面登録支援を入れる。

```text
OCR
項目抽出
kintone 自動入力候補
```

## フェーズ3

形状解析を足す。

```text
輪郭解析
ベクター解析
構造特徴量
```

---

# 優先順位

```text
1. OCR
2. 項目抽出
3. OCR検索
4. OpenCLIPの補助利用
5. 形状解析
6. ベクター解析
```

UI の整備は、類似検索がしっかり動くことを確認してからでよい。
