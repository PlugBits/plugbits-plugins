# kintone Drawing Similarity：現状と今後のロードマップ

最終更新: 2026-07-01

---

## 目的

kintone に登録された図面 PDF を対象に、類似図面検索と図面登録支援を実装する。

類似検索は画像 Embedding だけでなく以下を組み合わせる。

- 画像 Embedding 検索（DINOv2 / OpenCLIP）
- OCR 文字抽出
- 図番・品名・材質・寸法などの項目抽出
- kintone 項目・抽出項目による絞り込み／スコア補正
- 形状特徴量（shape features）によるスコア補正

---

## システム構成

```
kintone Plugin
  ↓  REST API (X-API-Key)
Cloud Run API（GCP: drawing-similarity-501101）
  ↓                  ↓
Qdrant Cloud     Firestore（テナント認証）
                 Vertex AI（OCR / Gemini）
```

### Cloud Run API（services/drawing-similarity-api）

| エンドポイント | 機能 |
|---|---|
| GET /health | 死活確認 |
| POST /index | PDF 取得 → PNG 化 → Embedding → OCR → Qdrant upsert。pdf_base64 直接受信に対応（プラグインがブラウザのkintoneセッションでファイルを取得して送る。サーバー側のkintone接続は不要になった）。fileKeyは整合性チェック用に引き続き必須。pdf_base64未指定時のみサーバー側kintone接続にフォールバック（旧プラグイン互換）。一括登録は Content-Type: application/octet-stream でPDFの生バイト列をボディに、メタデータを x-index-meta ヘッダーに載せるバイナリ直送経路にも対応（base64/JSON.stringifyの巨大な一時文字列割り当てを無くし、数千件連続処理でのブラウザOut of Memoryを防ぐ） |
| GET /thumbnail | kintoneのfileKeyからサーバー側でPDFを取得しPNG化して返す（レガシー。旧プラグイン互換用で、サーバーのKINTONE_BASE_URL/KINTONE_API_TOKEN設定が必須。新プラグインはクライアント取得＋POST /render-thumbnail を使うため呼ばれない） |
| POST /archive-index | kintone未登録の過去図面PDF（pdf_base64直接受信）を Qdrant に取り込み（doc_type='archive'、OCRはTesseract固定、サムネイル生成なし・kintone書き込みなし。Google Drive由来の場合はdriveFileIdも保存） |
| GET /google/oauth/popup | Google Drive連携用のOAuth同意＋Picker選択ポップアップ（当方サーバーがホストする固定オリジンページ。drive.fileスコープの一時アクセストークンのみでサーバー側の永続化なし） |
| GET /archive-status | テナント内の過去図面アーカイブ取込済み一覧（doc_type='archive'のみ。/index-statusは意図的にアーカイブを除外しているため別エンドポイント） |
| POST /render-thumbnail | 受け取ったPDFの1ページ目をPNG化して返すだけの状態を持たないユーティリティ（Google Driveアーカイブのサムネイル表示用。永続化・Drive認証には関与しない）。任意項目 max_width（16〜2000、既定はTHUMBNAIL_MAX_WIDTH）で出力幅を指定可能。TIFプレビュー表示など既定サムネイルより大きい画像が欲しい呼び出し元向け |
| POST /similar | Qdrant 類似検索 → スコアキャリブレーション → 返却（recordId / fileKey / pdf_base64 いずれのクエリにも対応、結果は docType='drawing'\|'archive' を明示） |
| POST /analyze | PDF → OCR → 項目抽出（kintone に保存しない） |
| POST /tag | タグ同期（Qdrant payload 更新） |
| POST /delete | レコード削除時のポイント削除（孤児ポイント対策） |
| GET /index-status | テナント内の登録済み record_id + file_key 一覧（整合性チェック用） |
| GET /tags | テナント内タグ一覧取得 |
| GET /field-values | テナント内フィールド値サジェスト取得 |

| 機能 | 備考 |
|---|---|
| DINOv2 埋め込み（現在使用中） | EMBEDDING_PROVIDER=dinov2 |
| OpenCLIP 埋め込み | EMBEDDING_PROVIDER=openclip |
| Tesseract OCR | 日本語＋英語 |
| 形状解析（shape features） | bbox・輪郭・ink ratio など |
| スコアキャリブレーション | vector floor/ceiling + metadata + shape の重み合成 |
| Firestore テナント認証基盤 | TENANT_AUTH_ENABLED で有効化 |
| テナント分離 Phase 2 | API キーから tenantId を解決し client 送信値を無視。Qdrant point ID を tenantId で名前空間化（衝突防止）。テナントドキュメントに `tenantId` フィールドが必要 |
| セキュリティ強化 | ①リクエストボディ上限 30MB（413） ②サムネイルは HMAC トークン（/similar が同梱・日単位で安定＝キャッシュ有効） ③Firestore ドキュメントID = SHA-256(APIキー) 推奨（生キーIDはレガシー互換） ④/health は無認証には最小情報のみ |

### 機能追加：手元の図面でその場検索（一覧画面）【実装済み】

**概要**

kintone に登録せずに、手元にある PDF をアップロードしてその場で類似図面を検索できる機能。

**ユースケース**

- 見積依頼を受けた図面が既存図面と似ているか素早く確認したい
- kintone に登録する前に類似品を調べたい

**想定フロー**

```
一覧画面「類似検索（アップロード）」ボタン
  ↓
PDF drag & drop モーダル
  ↓
/similar に pdf_base64 を直接送信（recordId 不要）
  ↓
類似結果をモーダルに表示（既存の類似検索 UI を流用）
```

**実装内容**

- plugin.js：一覧画面に「類似検索（アップロード）」ボタンを追加。`openUploadSimilarModal` で drag & drop → base64 化 → `/similar` 呼び出し → 既存の `renderSimilarList` で結果表示。
- server.js：`/similar` に `pdf_base64` パスを追加（recordId なしでその場で PNG 化 → shape profile + Embedding を生成して検索、Qdrant には保存しない。`queryVectorSource = 'uploaded-pdf'`）。
