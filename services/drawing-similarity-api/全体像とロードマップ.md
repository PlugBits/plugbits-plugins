kintone Drawing Similarity：全体像とロードマップ
目的

kintoneに登録された図面PDFを対象に、ズメーンのような 類似図面検索 を実装する。

ここでいう類似検索は、画像だけの検索ではなく、以下を含む。

画像Embedding検索
+
OCR文字抽出
+
図番・品名・材質・板厚・客先・改訂などの項目抽出
+
kintone項目・抽出項目による絞り込み/スコア補正

環境変数

Cloud Runに設定する。

NODE_ENV=production
QDRANT_URL=https://your-qdrant-url
QDRANT_API_KEY=your-api-key
QDRANT_COLLECTION=drawing_similarity
VECTOR_SIZE=384

将来追加候補：

OCR_ENGINE=paddleocr
OPENCLIP_MODEL=ViT-B-32
OPENCLIP_PRETRAINED=laion2b_s34b_b79k
データ設計

collection: drawing_similarity

完成形の処理フロー
/index
1. kintoneレコード情報を受け取る
2. kintone添付PDFを取得する
3. PDF 1ページ目をPNG化する
4. OpenCLIPで画像Embeddingを作成する
5. OCRで文字を抽出する
6. OCR結果から図番・品名・材質・板厚・客先・改訂を抽出する
7. kintone既存項目と抽出項目を統合する
8. Qdrantへ vector + payload をupsertする
/similar
1. tenant_id + record_id でQdrantから対象pointを取得する
2. 対象pointのvectorでQdrant検索する
3. tenant_idで必ずフィルタする
4. 自分自身のrecord_idを除外する
5. payload条件で補正する
   - 材質一致
   - 板厚近似
   - 客先一致
   - 品名/分類一致
6. 上位候補をkintoneへ返す
ロードマップ
Step 1：OpenCLIP実ベクトル化

現在のダミーベクトルをOpenCLIPに差し替える。

完了条件：

・同じPDFなら同じ/近いvectorになる
・似た形状の図面が上位に出る
・QdrantのVECTOR_SIZEとモデル出力次元が一致する
Step 2：OCR実装

PDF画像から文字を抽出する。

候補：

PaddleOCR
Tesseract
Cloud Vision API

まずはローカル/コンテナで使いやすいものを選定。

完了条件：

・図番らしき文字を取れる
・品名らしき文字を取れる
・材質らしき文字を取れる
・OCR全文をQdrant payloadに保存できる
Step 3：項目抽出

OCR全文から構造化項目を作る。

対象：

drawing_no
part_name
material
thickness
customer
revision
shape_category

最初はAIなしのルール抽出でよい。

・正規表現
・キーワード周辺抽出
・既存kintone項目優先
・OCR項目は補助扱い

完了条件：

・抽出結果をpayloadに保存できる
・抽出失敗時も/indexが落ちない
・confidenceを持てる
Step 4：スコアリング改善

単純なvector similarityだけでなく、payloadで補正する。

例：

final_score =
  vector_score
  + material_bonus
  + thickness_bonus
  + customer_bonus
  + category_bonus

注意：

・画像類似度を主軸にする
・材質/客先などは補正
・完全一致条件にしすぎない

完了条件：

・材質一致の候補が上がりやすい
・客先違いでも形状が近ければ残る
・検索結果が現場感覚に近くなる
Step 5：kintone UI改善

検索結果に以下を表示する。

・図番
・品名
・材質
・板厚
・客先
・類似度
・kintoneレコードリンク
・抽出元情報

サムネイルは類似検索UXに重要なので、ここで追加候補。

・PDF 1ページ目のPNGを作成
・小さいサムネイルを表示
・保存先は後で検討

完了条件：

・結果を見て現場が判断できる
・kintoneレコードへ遷移できる
・スコアだけでなく理由が分かる
Step 6：再インデックス機能

既存図面をまとめて登録できるようにする。

必要機能：

・現在レコードの再index
・一覧から複数件index
・未index件数の確認
・失敗ログ
・再実行

完了条件：

・100件程度を連続indexできる
・失敗しても処理が止まらない
・Qdrant points数を確認できる
類似検索完成の定義

以下を満たせば「類似検索完成」とする。

1. kintone詳細画面から図面登録/更新できる
2. PDFから画像Embeddingを作成できる
3. PDFからOCR文字抽出できる
4. 図番・品名・材質などを抽出できる
5. Qdrantにvector + payloadを保存できる
6. 類似図面検索が高速に返る
7. 検索結果から自分自身を除外できる
8. tenant_idでデータ分離できる
9. kintone画面で候補を確認できる
10. 30〜100枚程度で「似ている」と感じる結果が出る
