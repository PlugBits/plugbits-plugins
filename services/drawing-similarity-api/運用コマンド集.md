# 運用コマンド集（コピペ用）

日常の管理でよく使うコマンドを1枚にまとめたもの。詳しい背景・初回セットアップは
`テナント有効化手順.md` を参照。**特に断りがなければ Cloud Shell で実行する**
（PowerShell では動かないものが多い。PowerShell 用は最後の節にまとめた）。

## 共通の値

| 項目 | 値 |
|---|---|
| プロジェクト | `drawing-similarity-501101` |
| リージョン | `asia-northeast1` |
| Cloud Run サービス名 | `drawing-similarity-api` |
| サービスURL | `https://drawing-similarity-api-939943665629.asia-northeast1.run.app` |
| Firestore データベースID | `default`（括弧なし） |
| Firestore コレクション | `tenants` |
| Qdrant コレクション | `drawing_similarity_dinov2_base`（全テナント共通・payloadの`tenant_id`で分離） |

---

## 1. サーバーのデプロイ（コードを更新したとき）

```sh
cd ~/plugbits-plugins && git pull origin main
cd services/drawing-similarity-api
gcloud run deploy drawing-similarity-api \
  --source . \
  --region asia-northeast1 --project drawing-similarity-501101 \
  --clear-base-image
```

- サービス名 `drawing-similarity-api` を**必ず明示**（省くと別サービスが増える）
- 必ず `services/drawing-similarity-api`（Dockerfileのある場所）で実行
- 赤い `Regional Access Boundary ...` はノイズ。最後に `has been deployed` が出れば成功
- **プラグインのみの変更ならデプロイ不要**（kintoneへの再アップロードだけ）

## 2. テナント管理

### 新規テナント発行（1コマンド）

```sh
cd ~/plugbits-plugins/services/drawing-similarity-api
./scripts/provision-tenant.sh 新テナントのkintoneサブドメイン
```

出力された「APIキー」をそのテナントのプラグイン設定に貼る → 接続テスト。
`Permission denied` なら `bash scripts/provision-tenant.sh ...` で実行。

### キーが正しく通るかの検証（200なら成功）

```sh
curl -sSi 'https://drawing-similarity-api-939943665629.asia-northeast1.run.app/tags?tenantId=サブドメイン' \
  -H "X-API-Key: そのキー" | head -1
```

### テナントを無効化する（キーを止める）

Firestore コンソール → `default` DB → `tenants` → 該当ドキュメント →
`active` を `false` に変更（boolean のまま）。※最大60秒キャッシュされる。

### Firestoreのテナントドキュメントを直接確認（401の切り分け）

```sh
TOKEN=$(gcloud auth print-access-token)
HASH=$(printf %s "プラグインのキー" | sha256sum | cut -d' ' -f1)
curl -sS -H "Authorization: Bearer $TOKEN" \
  "https://firestore.googleapis.com/v1/projects/drawing-similarity-501101/databases/default/documents/tenants/$HASH"
```

`NOT_FOUND` ならドキュメント側の問題（キーとハッシュのズレ or 未作成）。

## 3. 登録状況の確認

### テナントの登録済み件数（kintone登録図面）

```sh
curl -sS 'https://drawing-similarity-api-939943665629.asia-northeast1.run.app/index-status?tenantId=サブドメイン' \
  -H "X-API-Key: そのキー" | head -c 300
```

冒頭の `"count": ◯` が登録済み件数。
**画面で見るなら**: そのテナントのkintone → 管理 ▾ → インデックス状況チェック
（登録済み/未登録/要更新/孤児が数字で出る。5000件一括登録の進捗確認もこれ）。

### アーカイブ（kintone未登録の過去図面）の取込件数

```sh
curl -sS 'https://drawing-similarity-api-939943665629.asia-northeast1.run.app/archive-status?tenantId=サブドメイン' \
  -H "X-API-Key: そのキー" | head -c 300
```

画面で見るなら: 管理 ▾ → アーカイブ取込状況。

### Qdrant UIで特定テナントだけに絞る

コレクション `drawing_similarity_dinov2_base` → Points のフィルタ欄に:

```json
{ "must": [ { "key": "tenant_id", "match": { "value": "サブドメイン" } } ] }
```

## 4. 死活確認・ログ

### ヘルスチェック

```sh
# 誰でも可（認証ON時は {ok, service} だけ返るのが正常）
curl -sS https://drawing-similarity-api-939943665629.asia-northeast1.run.app/health
# キー付き（runtime情報まで見える）
curl -sS https://drawing-similarity-api-939943665629.asia-northeast1.run.app/health -H "X-API-Key: キー"
```

### サーバーログ（直近）

```sh
gcloud beta run services logs read drawing-similarity-api \
  --region asia-northeast1 --project drawing-similarity-501101 --limit 50
```

またはコンソール → Cloud Run → drawing-similarity-api → ログ タブ。
個々のリクエストの成否はブラウザ側の DevTools（F12）→ Network タブも有効。

## 5. Cloud Run の設定変更（env・インスタンス数）

**envやインスタンス数の変更はコードを再ビルドしない**（コード反映は必ず「1.」のdeploy）。

```sh
# 環境変数の追加・変更
gcloud run services update drawing-similarity-api \
  --update-env-vars KEY=VALUE \
  --region asia-northeast1 --project drawing-similarity-501101

# 環境変数の削除（例: kintone接続情報の撤去。新プラグイン全展開後に任意で）
gcloud run services update drawing-similarity-api \
  --remove-env-vars KINTONE_BASE_URL,KINTONE_API_TOKEN \
  --region asia-northeast1 --project drawing-similarity-501101

# デモ・本番前のwarm化（コールドスタート回避）
gcloud run services update drawing-similarity-api \
  --min-instances=1 --region asia-northeast1 --project drawing-similarity-501101

# 開発に戻す
gcloud run services update drawing-similarity-api \
  --min-instances=0 --region asia-northeast1 --project drawing-similarity-501101
```

主なenv: `TENANT_AUTH_ENABLED`（テナント認証） / `FIRESTORE_DATABASE_ID=default` /
`THUMB_TOKEN_SECRET` / `GOOGLE_OAUTH_CLIENT_ID` / `GOOGLE_PICKER_API_KEY` /
`GOOGLE_CLOUD_PROJECT_NUMBER`（Drive連携3点） /
`SIMILAR_SCORE_FLOOR`（類似なし警告の閾値、デフォルト0.9）。

## 6. トラブル時の定番切り分け

401 / 接続不可はほぼこの順で潰れる:

1. **デプロイ済みコードが古い** → 「1.」を再実行（CORSエラー「x-api-key is not allowed」もこれ）
2. **キーとハッシュのズレ** → 「2.」のFirestore直接確認。NOT_FOUNDならスクリプトで再発行が早い
3. **Firestore DB名** → env `FIRESTORE_DATABASE_ID=default` が入っているか
4. **サービス二重作成** → `gcloud run services list --project drawing-similarity-501101` で
   サービスが1つだけか確認。増えていたら不要な方を削除し、プラグインの
   API Base URL が正しい方を向いているか確認

## 7. PowerShell（Windowsローカル）で確認したいとき

`curl`/`grep`/`sh` は使えないので `Invoke-WebRequest` を使う:

```powershell
# キーの検証（StatusCode: 200 なら成功）
(Invoke-WebRequest -UseBasicParsing `
  -Uri 'https://drawing-similarity-api-939943665629.asia-northeast1.run.app/tags?tenantId=サブドメイン' `
  -Headers @{ 'X-API-Key' = 'そのキー' }).StatusCode

# 登録件数の確認
(Invoke-WebRequest -UseBasicParsing `
  -Uri 'https://drawing-similarity-api-939943665629.asia-northeast1.run.app/index-status?tenantId=サブドメイン' `
  -Headers @{ 'X-API-Key' = 'そのキー' }).Content.Substring(0,300)
```

テナント発行スクリプトや gcloud 系は Cloud Shell で実行すること。
