# Excel帳票一括生成プラグイン 設定手順書

この手順書は、Excel帳票一括生成プラグインをkintoneアプリに設定するための手順です。

設定の入口は次の2パターンに分かれます。

- アプリテンプレートを利用する場合: 帳票テンプレート管理用のアプリや基本フィールドが用意済みの前提です。
- アプリテンプレートを利用しない場合: 必要なフィールドやテンプレート管理方法を手動で準備します。

帳票テンプレートの取得方法を設定した後は、どちらの場合も「共通設定」に進みます。

## 1. 事前に決めること

設定を始める前に、次の内容を決めておきます。

| 項目 | 内容 |
| --- | --- |
| 出力対象アプリ | Excelを出力したいレコードが入っているkintoneアプリ |
| Excelテンプレート | 差し込みタグを入れた `.xlsx` ファイル |
| テンプレート取得方法 | 帳票テンプレートアプリ、出力対象アプリの添付ファイル、固定URLのいずれか |
| 差し込みタグ | Excel内に入れる `{{partNumber}}` などのタグ名 |
| 差し込み元フィールド | 各タグに対応するkintoneフィールド |
| 複数ラベル作成の有無 | 数量に応じて1レコードから複数シートを作るか |
| 印刷ステータス更新の有無 | 出力後にレコードのステータスや出力日時を更新するか |

## 2. Excelテンプレートを準備する

Excelファイルに、kintoneの値を差し込みたい場所をタグで入力します。

通常の文字差し込み:

```text
{{partNumber}}
{{qty}}
{{labelQty}}
{{seqNo}} / {{seqTotal}}
```

QR画像を出力する場合:

```text
{{QR:ticketQr}}
```

バーコード画像を出力する場合:

```text
{{BC:partNumber}}
```

QRやバーコードを置く場合は、配置したいセル範囲をExcel側で結合し、結合範囲の左上セルにタグを入力します。出力時に、その結合セル範囲内へ画像が配置されます。

## 3A. アプリテンプレートを利用する場合

アプリテンプレートを利用する場合は、帳票テンプレートアプリが用意されている前提で進めます。

### 3A-1. 帳票テンプレートアプリを確認する

帳票テンプレートアプリに、少なくとも次のフィールドがあることを確認します。

| 用途 | 標準フィールドコード | 内容 |
| --- | --- | --- |
| テンプレートコード | `templateCode` | 使用する帳票を識別するコード。例: `WI_LABEL` |
| テンプレートファイル | `templateFile` | Excelテンプレートの添付ファイル |
| ステータス | `status` | 有効なテンプレートかどうかを判定する値 |

ステータス値は、通常 `ACTIVE` を有効値として使います。

### 3A-2. 帳票テンプレートレコードを作成する

帳票テンプレートアプリで、使用するExcelテンプレート用のレコードを作成します。

1. `templateCode` にテンプレートコードを入力します。
   例: `WI_LABEL`
2. `templateFile` にExcelテンプレートファイルを添付します。
3. `status` に `ACTIVE` を入力または選択します。
4. レコードを保存します。

### 3A-3. 出力対象アプリでプラグイン設定を開く

1. 出力対象アプリの設定画面を開きます。
2. プラグイン設定から「Excel帳票一括生成」を開きます。
3. 「テンプレートの取得方法」で「帳票テンプレートアプリから取得」を選択します。
4. 次の項目を入力します。

| 設定項目 | 入力内容 |
| --- | --- |
| 帳票テンプレートアプリID | 帳票テンプレートアプリのアプリID |
| 使用するテンプレートコード | 帳票テンプレートレコードの `templateCode` の値 |
| テンプレート添付ファイルのフィールドコード | 通常は `templateFile` |
| テンプレートコードのフィールドコード | 通常は `templateCode` |
| ステータスのフィールドコード | 通常は `status` |
| 有効ステータス値 | 通常は `ACTIVE` |

入力できたら「4. 共通設定」に進みます。

## 3B. アプリテンプレートを利用しない場合

アプリテンプレートを利用しない場合は、テンプレートの置き場所を手動で決めます。

おすすめは、帳票テンプレートアプリを手動で作成し、「帳票テンプレートアプリから取得」を使う方法です。複数の帳票を管理しやすく、出力対象アプリにテンプレートファイルを添付しなくて済みます。

### 3B-1. 方法1: 帳票テンプレートアプリを手動で作る

帳票テンプレート管理用のkintoneアプリを作成し、次のフィールドを追加します。

| フィールド名の例 | フィールド種類 | フィールドコード |
| --- | --- | --- |
| テンプレートコード | 文字列1行 | `templateCode` |
| テンプレートファイル | 添付ファイル | `templateFile` |
| ステータス | ドロップダウンまたは文字列1行 | `status` |

その後、帳票テンプレートレコードを作成します。

1. `templateCode` に任意のコードを入力します。
   例: `WI_LABEL`
2. `templateFile` にExcelテンプレートを添付します。
3. `status` を `ACTIVE` にします。
4. レコードを保存します。

出力対象アプリのプラグイン設定では、「帳票テンプレートアプリから取得」を選択し、3A-3と同じ項目を入力します。

### 3B-2. 方法2: 出力対象アプリの添付ファイルから取得する

出力対象アプリの各レコードにExcelテンプレートを添付して使う方法です。

1. 出力対象アプリに添付ファイルフィールドを追加します。
2. 各出力対象レコードにExcelテンプレートを添付します。
3. プラグイン設定の「テンプレートの取得方法」で「このアプリの添付ファイルから取得」を選択します。
4. 「テンプレート添付ファイル」で、追加した添付ファイルフィールドを選択します。

この方法では、出力対象として選択したレコードの添付ファイルからテンプレートを取得します。レコードごとに異なるテンプレートを使いたい場合に向いています。

### 3B-3. 方法3: 固定URLから取得する

テンプレートファイルをURLで取得する方法です。

1. Excelテンプレートを取得可能なURLに配置します。
2. プラグイン設定の「テンプレートの取得方法」で「固定URLから取得」を選択します。
3. 「テンプレートURL」に `.xlsx` ファイルのURLを入力します。

ブラウザから取得できるURLである必要があります。アクセス権限やCORSの制限がある場合、テンプレート取得に失敗することがあります。

## 4. 共通設定

ここから先は、アプリテンプレート利用時も、利用しない場合も同じです。

### 4-0. 出力モード

出力モードを選択します。

| モード | 用途 |
| --- | --- |
| ラベル出力 | 既存の1レコード1シート、または数量に応じた複数シート出力 |
| 帳票出力 | サブテーブル明細を指定行数ごとに分け、ページごとに別シートへ出力 |

帳票出力では、Excelテンプレートのヘッダー部分には通常の `{{orderNo}}` などのタグを使えます。ページ番号は `{{pageNo}}`、総ページ数は `{{pageTotal}}` を使えます。

明細は、テンプレート内にタグを書くのではなく、設定画面で次の内容を指定します。

| 設定項目 | 内容 |
| --- | --- |
| 明細サブテーブル | 明細として出力するkintoneサブテーブル |
| 明細開始行 | Excelテンプレート上で明細を書き始める行番号 |
| 1ページあたりの明細行数 | この行数を超えた明細は次のシートへ出力 |
| 余り行の表示 | 明細がない空行を非表示にするか |
| 明細列設定 | Excel列とサブテーブル内フィールドの対応 |

例:

| Excel列 | サブテーブル内フィールド |
| --- | --- |
| `A` | 品番 |
| `B` | 品名 |
| `C` | 数量 |

明細が30行あり、1ページあたりの明細行数を20にした場合、1シート目に1-20行目、2シート目に21-30行目が出力されます。

「明細がない余り行を非表示にする」をオンにすると、最後のページなどで空の明細行が隠れます。合計欄や備考欄など、明細行より下にある行は上に詰まります。

### 4-1. ラベル枚数設定

1レコードから1シートだけ作る場合は、「数量に応じてラベルを複数枚作成する」をオフのままにします。

数量に応じて複数シートを作る場合は、オンにして次の項目を設定します。

| 設定項目 | 内容 |
| --- | --- |
| 全体数量のフィールド | 総数量が入っているフィールド |
| 1ラベルあたりの数量フィールド | 1枚あたりの数量が入っているフィールド |
| ラベル数量タグ名 | 通常は `labelQty` |

例:

- 全体数量が `100`
- 1ラベルあたりの数量が `25`
- 出力シート数は `4`
- 各シートの `{{labelQty}}` は `25`

端数がある場合:

- 全体数量が `110`
- 1ラベルあたりの数量が `25`
- 出力シート数は `5`
- 最後のシートの `{{labelQty}}` は `10`

複数ラベル作成時に使える主な自動生成値は次の通りです。

| タグ | 内容 |
| --- | --- |
| `{{labelQty}}` | そのシートに出力する数量 |
| `{{seqNo}}` | 何枚目か |
| `{{seqTotal}}` | 全部で何枚か |
| `{{splitKey}}` | レコードIDと連番から作る分割キー |

### 4-2. 印刷ステータス更新

Excel出力後にレコードを更新したい場合は、「印刷ステータス更新」をオンにします。

| 設定項目 | 内容 |
| --- | --- |
| 印刷ステータスフィールド | 出力済み状態を保存するフィールド |
| 初回印刷ステータス値 | 未出力などから出力したときに入れる値。標準は `PRINTED` |
| 再印刷ステータス値 | すでに出力済みのレコードを再出力したときに入れる値。標準は `REPRINTED` |
| 印刷日時フィールド | 出力日時を保存するフィールド。不要なら未選択 |
| 印刷者フィールド | 出力者を保存するフィールド。不要なら未選択 |

現在値が `PRINTED` または `REPRINTED` の場合は、次回出力時に `REPRINTED` が入ります。それ以外の場合は `PRINTED` が入ります。

### 4-3. QR画像

QR画像を使う場合は、「QR画像を出力する」をオンにします。

Excel側では、QRを置きたい結合セルの左上セルに次のように入力します。

```text
{{QR:ticketQr}}
```

`ticketQr` の部分は差し込みタグ名です。差し込み設定で `ticketQr` に値が入るようにしてください。

「QR余白(px)」は通常 `4` のままで問題ありません。画像が窮屈な場合は少し大きくし、画像を大きくしたい場合は小さくします。

### 4-4. Excel差し込み設定

「Excel内のタグ名」と「差し込む値」を対応付けます。

1. 「差し込み項目を追加」を押します。
2. 「Excel内のタグ名」に、Excel側の `{{ }}` の中身を入力します。
   例: Excelに `{{partNumber}}` と書いた場合は `partNumber`
3. 「差し込む値の種類」を選びます。
4. 「選択内容」を設定します。

差し込む値の種類は次の3つです。

| 種類 | 使い方 |
| --- | --- |
| kintoneフィールド | 出力対象レコードのフィールド値を差し込む |
| 自動生成値 | レコードID、連番、ラベル数量、日付などを差し込む |
| 固定文字 | 常に同じ文字を差し込む |

設定例:

| Excel内のタグ名 | 種類 | 選択内容 |
| --- | --- | --- |
| `partNumber` | kintoneフィールド | 品番フィールド |
| `qty` | kintoneフィールド | 数量フィールド |
| `labelQty` | 自動生成値 | ラベル数量 |
| `seqNo` | 自動生成値 | ラベル番号 |
| `seqTotal` | 自動生成値 | ラベル総数 |
| `ticketQr` | 自動生成値またはkintoneフィールド | QRにしたい値 |

主な自動生成値:

| 値 | 内容 |
| --- | --- |
| `recordId` | kintoneレコード番号 |
| `seqNo` | ラベル番号 |
| `seqTotal` | ラベル総数 |
| `labelQty` | ラベル数量 |
| `splitIndex` | 0始まりの分割番号 |
| `splitKey` | レコードIDと連番のキー |
| `ticketId` | チケットID。未設定時はレコードID |
| `qrText` | QR文字列。未設定時はチケットIDなど |
| `ticketQr` | チケットQR用の文字列 |
| `partQr` | 品番QR用の文字列 |
| `today` | 今日の日付 |
| `now` | 現在日時 |

### 4-5. 出力・表示設定

| 設定項目 | 内容 |
| --- | --- |
| 出力ファイル名 | ダウンロードされるExcelファイル名。空の場合は `TicketLabels_YYYYMMDD_HHmm.xlsx` |
| 最大シート数 | 1回の出力で作成できる最大シート数。標準は `100` |
| 一覧画面のボタン名 | kintone一覧画面に表示するボタン名。空の場合は「ラベルを生成」 |
| 横幅補正 | Excel出力後に列幅がずれる場合の補正値。通常は `100%` |
| 縦高さ補正 | Excel出力後に行高さがずれる場合の補正値。通常は `100%` |
| ボタンを表示する一覧 | 未選択の場合はすべての一覧に表示 |

Excelの見た目が少しずれる場合だけ、横幅補正や縦高さ補正を調整します。

## 5. 保存とアプリ更新

1. プラグイン設定画面で「保存」を押します。
2. アプリ設定画面に戻ります。
3. kintoneアプリの「アプリを更新」を実行します。

保存しただけでは、本番画面に反映されない場合があります。必ずアプリ更新まで行ってください。

## 6. 動作確認

1. 出力対象アプリの一覧画面を開きます。
2. プラグインのボタンが表示されていることを確認します。
3. 出力したいレコードを含む一覧を表示します。
4. ボタンを押します。
5. 出力対象レコードを選択します。
6. `xlsx生成` を押します。
7. Excelファイルがダウンロードされることを確認します。
8. 差し込み値、QR、バーコード、シート数、ファイル名を確認します。
9. 印刷ステータス更新を有効にした場合は、対象レコードが更新されていることを確認します。

## 7. よくある確認ポイント

| 症状 | 確認すること |
| --- | --- |
| ボタンが表示されない | テンプレート取得方法の必須項目が入っているか、表示対象一覧が限定されていないか、アプリ更新済みか |
| テンプレートが見つからない | 帳票テンプレートアプリID、テンプレートコード、ステータス値、添付ファイルフィールドコードを確認 |
| 差し込みされない | Excelのタグ名と設定画面のタグ名が一致しているか確認。`{{partNumber}}` なら設定は `partNumber` |
| QRが出ない | `{{QR:タグ名}}` のタグに値が入っているか、QR画像がオンか確認 |
| バーコードが出ない | `{{BC:タグ名}}` のタグに値が入っているか確認。長すぎる文字列や未対応文字に注意 |
| 複数ラベルが作れない | 全体数量と1ラベルあたり数量が正の数として読み取れるか確認 |
| 最大シート数エラーになる | 選択レコード数と複数ラベル作成後のシート数が最大シート数を超えていないか確認 |
| 印刷ステータスが更新されない | 印刷ステータスフィールド、更新先の値、フィールド権限を確認 |

## 8. 最小設定例

帳票テンプレートアプリを使う場合の最小構成です。

帳票テンプレートアプリ:

| フィールドコード | 値 |
| --- | --- |
| `templateCode` | `WI_LABEL` |
| `templateFile` | Excelテンプレートを添付 |
| `status` | `ACTIVE` |

出力対象アプリのプラグイン設定:

| 設定項目 | 値 |
| --- | --- |
| テンプレートの取得方法 | 帳票テンプレートアプリから取得 |
| 帳票テンプレートアプリID | 帳票テンプレートアプリのID |
| 使用するテンプレートコード | `WI_LABEL` |
| テンプレート添付ファイルのフィールドコード | `templateFile` |
| テンプレートコードのフィールドコード | `templateCode` |
| ステータスのフィールドコード | `status` |
| 有効ステータス値 | `ACTIVE` |

Excelテンプレート:

```text
品番: {{partNumber}}
数量: {{labelQty}}
QR: {{QR:ticketQr}}
```

差し込み設定:

| Excel内のタグ名 | 種類 | 選択内容 |
| --- | --- | --- |
| `partNumber` | kintoneフィールド | 品番フィールド |
| `labelQty` | 自動生成値 | ラベル数量 |
| `ticketQr` | 自動生成値 | ticketQr |
